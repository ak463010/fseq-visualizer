# Contributing to FSEQ Analysis Toolkit

We welcome contributions from everyone! Whether you’re fixing a bug, adding a new feature, improving documentation, or sharing ideas, your input helps make this project better.

## How to Contribute

### 1. Reporting Issues
- Use the **Issues** tab to report bugs, request features, or suggest improvements.
- Provide clear details, including steps to reproduce (if applicable).

### 2. Submitting Changes
- Fork the repository.
- Create a new branch for your work (`git checkout -b feature-name`).
- Make your changes and commit them with descriptive messages.
- Push your branch and open a **Pull Request**.

### 3. Code Guidelines
- Keep code clean, readable, and well-documented.
- Follow existing project structure and naming conventions.
- Add comments where necessary to explain complex logic.

### 4. Documentation
- Update README.md or other docs if your changes affect usage.
- Ensure instructions are clear and concise.

### 5. Collaboration
- Be respectful and constructive in discussions.
- Review other contributions and provide helpful feedback.

## Community Values
- **Open Collaboration**: Everyone is welcome to participate.
- **Transparency**: Share ideas and decisions openly.
- **Respect**: Treat all contributors with kindness and professionalism.

## License Reminder
By contributing, you agree that your work will be distributed under the same free and open license as the project.
